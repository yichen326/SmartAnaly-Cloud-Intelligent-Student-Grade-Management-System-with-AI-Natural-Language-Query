## 1. 项目概述

**智析云途**是一款基于 **PyQt5 图形界面 + DeepSeek 大语言模型 API + SQLite3 数据库** 的智能学生成绩管理系统，专为中小学成绩管理场景设计。系统不仅提供传统的增删改查和排名功能，更创新性地引入 **AI自然语言交互引擎**，让用户可以通过日常对话式的自然语言完成复杂的数据查询、成绩分析和学习评估。

### 1.1 核心特性

- 🎯 **自然语言交互**：支持日常对话、复合问题、比较分析等复杂场景
- 📊 **成绩全流程管理**：录入、编辑、查询、排名、导出
- 🤖 **AI智能分析**：DeepSeek V4 Flash 大模型驱动的班级/年级/学生评估
- 🏆 **三级排名系统**：班级排名、年级排名、全校排名自动更新
- 🔐 **严谨数据校验**：8位学号严格格式验证，重复检测
- 🎨 **现代化GUI**：渐变主题、响应式布局、上下文菜单

### 1.2 适用场景

- 中小学班主任、年级主任日常成绩管理
- 学校教务处的数据统计与分析
- 学生个人学习情况评估与跟踪
- 教学质量的班级/年级横向对比

## 4. 核心模块详解

### 4.1 DeepSeekAI — AI API调用封装

```python
class DeepSeekAI:
    def __init__(self, api_key):      # API密钥初始化
    def chat(self, messages, temp):    # HTTP POST调用DeepSeek API
    def get_system_prompt(self, ctx):  # 构建AI角色指令（prompt工程）
    def build_context(self, db):       # 从数据库构建上下文数据
```

**核心设计思路**：

1. **Model Agnostic**：模型名称通过 `config.json` 的 `model` 字段配置，当前使用 `deepseek-v4-flash`，可随时切换
2. **Context as Data**：`build_context()` 不只是传空prompt，而是将每位学生的**详细成绩、等级、排名**都嵌入到 system prompt 中，让AI基于真实数据回答
3. **Prompt Engineering**：`get_system_prompt()` 包含完整的【FUNC】规则、场景示例、回答策略，确保AI行为可控

**`build_context()` 构建的数据上下文示例**：
```
本校共有 150 名学生，分布在以下年级和班级：
  高一1班：50人，均分74.5
  高一2班：50人，均分78.2
  ...
全校总分范围: 180 - 570，平均分: 375.2
各科成绩概况：
  语文：均分68.5，范围0-98
  ...
高三前三名：张三(568分)、李四(552分)、...

【每位学生详细成绩】
  张三(高三1班)：语文85，数学92，英语88，物理78，化学90，生物85，总分518，班排#1，年排#1
  ...
```

**这种设计使得AI可以**：
- 直接回答 "张三和李四谁成绩好"（不触发FUNC，在context中已有数据）
- 直接回答 "哪科平均分最高"（各科均分已在context中）
- 直接回答 "哪些同学是优秀等级"（每位学生的等级可推断）

### 4.2 AIChatEngine — AI意图解析引擎

```
用户输入
    │
    ▼
try_direct_match()
    │  ├── 菜单？ → 返回菜单文本
    │  ├── 退出？ → 返回退出消息
    │  └── 添加指令？ → 返回格式提示
    │
    └── 未匹配
          │
          ▼
_parse_intent_with_ai()
    │
    ├── 构建包含完整数据的 system prompt
    ├── 调用 DeepSeekAI.chat()
    │
    ├── 响应含 [FUNC] → 解析功能代号 + 参数映射 → 返回 (AI文本, 功能, 参数)
    │
    └── 响应不含 [FUNC] → 返回 (AI文本, "ai_response", None)
```

**关键设计决策**：

**1. 双路径路由策略**
- **快速路径** (try_direct_match)：菜单、退出等无需AI的指令，零延迟响应
- **AI路径** (_parse_intent_with_ai)：所有自然语言输入都走AI，确保灵活性和智能性

**2. AI vs 本地功能的智能分工**
- AI负责**判断何时触发功能**，而不是执行功能本身
- 触发 [FUNC] 后，系统执行功能并将数据返回给用户
- 不触发 [FUNC] 时，AI直接基于context中的数据回答

**3. 参数映射桥接** (`_map_func_params`)
- AI返回的是字符串（如 "高一"、"1班"）
- 系统需要的是标准代码（如 "01"、"01"）
- 映射器负责：年级名称→代码、班级编号补零、格式清理

### 4.3 GradeEvaluator — 成绩评估引擎

**评估维度**：

| 维度 | 方法 | 输出 |
|------|------|------|
| 单科等级 | `get_grade_level(score)` | 优秀/良好/中等/及格/不及格 |
| 学生评估 | `evaluate_student(student)` | 各科详情、优势科目、待加强科目、评语 |
| 班级评估 | `evaluate_class(db, gc, cc)` | 均分、各科均分、等级分布、及格率、最高/最低分 |
| 年级评估 | `evaluate_grade(db, gc)` | 班级对比、年级均分、差距分析 |

**等级划分标准**：
```
优秀：90-100分
良好：80-89分
中等：70-79分
及格：60-69分
不及格：0-59分
```

### 4.4 Database — 数据访问层

**设计特点**：
1. **自动建表**：`__init__` 中调用 `_create_tables()`，首次运行自动建表
2. **线程安全**：`check_same_thread=False` 允许跨线程访问
3. **Row工厂模式**：`row_factory = sqlite3.Row` 支持列名访问
4. **TXT数据导入**：`load_from_txt()` 支持标准格式文件批量导入，返回详细的导入统计（成功数、跳过数、错误列表）
5. **事务封装**：每次 `execute()` 自动 commit

**表结构**：
```sql
students (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    student_id TEXT UNIQUE NOT NULL,   -- 8位GGCCNNNN
    name TEXT NOT NULL,
    gender TEXT CHECK(gender IN ('男','女')),
    grade_code TEXT NOT NULL,          -- 01/02/03
    class_code TEXT NOT NULL,          -- 01-30
    chinese/math/english/physics/chemistry/biology REAL DEFAULT 0,
    total_score REAL DEFAULT 0,
    class_rank INTEGER DEFAULT -1,
    grade_rank INTEGER DEFAULT -1,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
)
```

### 4.5 排名系统

**更新算法**：
```python
def update_all_ranks(self):
    # 1. 班级排名：按 grade_code + class_code 分组，按 total_score DESC 排序
    class_groups = fetchall("SELECT DISTINCT grade_code, class_code FROM students")
    for group in class_groups:
        rows = fetchall("SELECT id FROM students WHERE grade_code=? AND class_code=? ORDER BY total_score DESC")
        for i, row in enumerate(rows, 1):
            UPDATE class_rank=i WHERE id=row.id
    
    # 2. 年级排名：按 grade_code 分组，按 total_score DESC 排序
    grade_groups = fetchall("SELECT DISTINCT grade_code FROM students")
    for group in grade_groups:
        rows = fetchall("SELECT id FROM students WHERE grade_code=? ORDER BY total_score DESC")
        for i, row in enumerate(rows, 1):
            UPDATE grade_rank=i WHERE id=row.id
```

**更新触发器**：添加、删除、修改学生 → 自动调用 `update_all_ranks()`

**性能优化**：对于大规模数据，可考虑使用 SQL 窗口函数 `RANK()` 替代逐行更新，但在中小规模数据（<1000人）下当前方案足够高效。

### 5.2 数据流分析

**场景：用户输入 "高一1班怎么样？高三2班呢？"**

```
步骤1: 用户输入 → MainWindow.send_message()
         │

         │
```
