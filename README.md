## 📖 项目简介

**智析云途** 是一款融合了传统数据库管理与 AI 大语言模型的学生成绩管理系统。它不仅提供了完整的学生信息管理、成绩录入/修改/删除、排名统计等基础功能，还接入了**火山方舟豆包大模型**，支持自然语言交互、智能数据分析以及人性化的情感点评。

无论你是直接输入指令（如"显示所有学生"），还是随意聊天（如"给点学习建议"），系统都能智能理解你的意图并给出恰当响应。

## ✨ 功能特性

### 📋 基础管理功能

| 功能 | 说明 | 示例指令 |
|------|------|----------|
| 学生列表 | 查看全部学生信息及成绩 | `显示所有学生` / `查看学生名单` |
| 添加学生 | 带学号格式验证的添加 | `添加学生 01020006 王小二 男 75 82 90 68` |

| 删除学生 | 支持模糊匹配删除 | `删除张三` |
| 导出 Excel | 导出完整的学生信息和成绩表 | `导出Excel` |

### 📊 排名统计

| 排名类型 | 说明 | 示例指令 |
|----------|------|----------|
| **全校排名** | 按总分/单科进行全校排名 | `总分排名` / `语文排名` |
| **班级排名** | 按班级进行总分/单科排名 | `高一1班排名` / `1班数学排名` |
| **年级排名** | 按年级进行总分/单科排名 | `高一年级排名` / `高二年级英语排名` |

### 🤖 AI 智能分析

| 分析类型 | 说明 | 示例指令 |
|----------|------|----------|
| **班级分析** | 平均分、最高/最低分、及格率、优秀率 | `分析高一1班成绩` |
| **年级分析** | 年级整体情况、班级分布 | `分析高二年级` |
| **单科详细分析** | 优秀/良好/及格/不及格分段统计、名单 | `分析高一1班数学` |
| **自由分析** | 针对未匹配的分析需求调用AI智能回答 | `看看高一1班怎么样` |

### 💬 AI 交互

- **人性化点评**：每个操作执行后自动生成一句温暖的小评
- **自由聊天**：学习建议、情感交流、日常对话均可
- **自然语言理解**：模糊匹配、同义词识别、口语化表达

## 🔢 学号标准（GGCCNNNN）

学号为 **8位纯数字**，结构如下：

```
  G  G  C  C  N  N  N  N
 01  01  00  01
 │   │   │   └── 顺序号（0001~9999）
 │   │   └────── 班级编号（01~30）
 │   └────────── 年级代码（01=高一, 02=高二, 03=高三）
 └────────────── 固定格式
```

**示例：**
- `01010001` → 高一1班第1号
- `02020015` → 高二2班第15号
- `03030099` → 高三3班第99号

**验证规则：**
- ✅ 必须为8位纯数字
- ✅ 年级代码只能是 `01` `02` `03`
- ✅ 班级编号范围 `01` ~ `30`
- ✅ 顺序号范围 `0001` ~ `9999`
- ✅ 学号不可重复

## 🔧 开发者指南

### 架构设计要点

1. **单文件设计**：为便于快速部署和教学演示，所有代码集中在 `main.py`。正式生产环境建议拆分。

2. **AI 双模式**：
   - **模式一（固定指令）**：正则 + 条件分支精确匹配，保证核心功能 100% 可靠
   - **模式二（AI 自由对话）**：未匹配的输入交给大模型自由发挥，保证灵活性

3. **线程安全**：AI API 调用使用 `QRunnable` + `QThreadPool` 异步执行，避免阻塞 UI

4. **学号即主键**：以 `sid`（学号）作为 `student` 表和 `score` 表的主键，通过 JOIN 关联

### 扩展建议

如果你希望改进或扩展该项目，以下是推荐的优化方向：

| 方向 | 具体建议 | 涉及文件/模块 |
|------|----------|---------------|
| **多用户系统** | 添加用户注册、角色权限（管理员/教师/学生） | Database 类、LoginDialog |
| **UI 美化** | 使用 QSS 样式表或 PyQt5 的 QGraphicsDropShadowEffect | MainWindow.init_ui |
| **更多科目** | 在 score 表和所有 SQL 查询中添加新字段 | Database 类全局 |
| **图表可视化** | 集成 matplotlib 或 pyqtgraph 绘制柱状图/饼图 | AIWorker 或新建模块 |
| **批量导入** | 支持从 CSV/Excel 批量导入学生数据 | Database 类 |
| **分页查询** | 学生较多时分页显示 | get_all_table 方法 |
| **配置分离** | 将 API Key、EP_ID、数据库路径等放入 config.json | 新建 config.py |
| **错误日志** | 使用 logging 模块记录操作日志和错误日志 | 全局 |
| **单元测试** | 使用 pytest 为 Database 和 DoubaoAI 编写测试 | 新建 tests/ |
| **Web 版本** | 使用 Flask/FastAPI 重构为 Web 应用 | 整体重构 |
| **成绩趋势** | 存储历次考试成绩，显示提分/退步趋势 | score 表增加时间字段 |
| **通知推送** | 集成钉钉/企业微信 Bot，推送成绩异常通知 | 新建模块 |
| **语音交互** | 集成语音识别（如百度语音 API） | MainWindow |
| **多语言支持** | UI 文本抽离为国际化资源文件 | MainWindow |

## 🔑 API 配置

系统默认使用 **火山方舟豆包大模型**。如需更换或配置自己的 API：

在 `main.py` 顶部找到以下配置项：

```python
ARK_API_KEY = "your-api-key-here"
EP_ID = "your-endpoint-id-here"
ARK_URL = "https://ark.cn-beijing.volces.com/api/v3/chat/completions"
```

> **注意**：为安全起见，生产环境建议将 API Key 通过环境变量读取，而非硬编码在代码中。
